# Задача 4 - разбор проблемного Dockerfile

## Исходный Dockerfile

```dockerfile
FROM ubuntu:latest
MAINTAINER MyCompany

COPY . /var/www/html
RUN apt-get update -y
RUN apt-get install -y nginx
CMD ["nginx", "-g", "daemon off;"]
````

## Что не так

### Общее замечание

* А точно нужно прям целую Ubuntu накатывать? Ну... представим, что нам недостаточно лёгких образов подобных ОС / официального образа nginx. Нам же не сказали специфику требований к контейнеру. Но вопросики есть в воздухе

### `FROM ubuntu:latest`

* `latest` - то, с чего начинается лекция по best practice в DOCKERFILE :) Кто сказал, что нам 18+ картинку вместо образа ОС в последней версии по ошибке не загрузят? Лучше на чёткую версию образа ссылаться, маловероятно, что его перезапишут случайно

### `MAINTAINER MyCompany`

* Загуглил (а то что-то отдалённо знакомое, но явно не знакомое), устарела с версии `Docker 1.13.0` в январе 2017. Для добавления метаданных на `LABEL` нужно переходить

### `COPY . /var/www/html`

* Копируется вообще всё из build context в /var/www/html (кто сказал, что тут не будет лишнего мусора, особенно тяжеловесного, или вообще чего хуже - чувствительных данных). 
* Вероятность воспользоваться закэшированным слоем при повторной сборке существенно ниже (так как выше вероятность изменения папки, чем отдельных файлов)
* В идеале - никогда не копипастить целыми папками (но часто мы можем не знать весь перечень файлов, так что идеал редко достигаем)

> Есть, конечно, оправдание такому, возможно, на хосте качественный `.dockerignore`, в котором всё запрещено, кроме разрешённого

### `RUN apt-get update -y`

* Плохо держать отдельно от `apt-get install`, больше слоёв кэша без надобности

### `RUN apt-get install -y nginx`

* Установка пакета вынесена в отдельный слой (см. прошлый раздел)
* Для легковестности, в теории, можно отказаться от установки необязательных зависимостей (--no-install-recommends; но я скептически отношусь к этой практике по умолчанию, нужно сначала тестировать, не крякнет ли прога от такого в нашей среде)
* Не очищается apt-кэш (образ становится тяжеловеснее)

## Исправленный вариант

```dockerfile
FROM ubuntu:24.04
LABEL maintainer="MyCompany"

RUN apt-get update \
    && apt-get install -y --no-install-recommends nginx \
    && rm -rf /var/lib/apt/lists/*
COPY ./бла-бла.txt /var/www/html/
CMD ["nginx", "-g", "daemon off;"]
```
